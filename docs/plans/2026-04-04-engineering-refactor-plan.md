# AccOps 工程化重构计划

> 目标：在不改变核心业务能力的前提下，降低前后端耦合、缩小超大文件、统一命名与仓库规范，并补上可自动执行的验证链路。

## 当前主要问题

1. 仓库产物混入源码树：`.venv`、`.browser_profiles`、`__pycache__`、`.DS_Store` 等运行产物位于项目目录内。
2. 前端依赖管理不统一：`package-lock.json` 与 `pnpm-lock.yaml` 并存，实际依赖树曾处于损坏状态。
3. 页面与领域逻辑耦合严重：`GroupDetail.tsx`、`AccountsPage.tsx`、`SmsPage.tsx`、`OperationPanel.tsx` 体量过大，重复逻辑较多。
4. 后端职责边界不清晰：路由层混入解析/同步业务，服务层同时承担序列化与持久化协作。
5. 自动化验证薄弱：后端没有测试入口，前端没有回归测试脚本，README 与真实入口不一致。

## 本次重构范围

### 仓库规范

- 统一前端包管理工具到 `pnpm`
- 补齐根级忽略规则并修正后端忽略规则错误
- 更新 README 中与实际实现不一致的入口/命令描述

### 前端重构

- 抽离自动化操作定义为共享模块，消除页面和组件中的重复定义
- 抽离浏览器状态管理与通用错误处理
- 统一页面命名和导入关系，修正不一致文件名
- 清理 `any`、未使用变量和不稳定 hook 依赖
- 增加前端单元测试，覆盖共享纯逻辑

### 后端重构

- 将账号导入解析从路由层抽离为独立模块
- 将通用序列化/辅助逻辑从服务主类中拆开，减少职责混杂
- 移除已失效或无用的依赖注入内容
- 增加后端单元测试，覆盖导入解析等核心纯逻辑

## 验证基线

### 前端

- `pnpm lint`
- `pnpm build`
- `pnpm test --run`

### 后端

- `uv run python -m unittest discover -s tests -p \"test_*.py\"`
- `uv run python -m compileall app.py config.py deps.py core models routers services utils`

## 风险控制

1. 优先提取纯逻辑和共享配置，再移动页面/组件命名，避免一次性大搬家。
2. 每次抽离都以测试或构建失败面为边界，先红后绿。
3. 对仓库内已有运行产物先通过忽略规则隔离；真正删除前再单独确认。
